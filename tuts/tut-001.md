# Tutorial 1 - Requisitos Básicos

Neste tutorial iremos desenvolver um Sistema Operacional Simples com a finalidade de auxiliar quem estiver iniciando seus estudos sobre o funcionamento e desenvolvimento de Sistemas Operacionais.

Para este projeto iremos considerar que se está utilizando um Sistema Operacional Linux como Host. Sistema Host é aquele sistema operacional que é utilizado pelo desenvolvedor para implementação e depuração de outro sistema operacional.

Para a implementação iremos utilizar as linguagens C e ASM para o desenvolvimento do sistema.

A primeira coisa a fazer, além de se buscar conhecimento sobre o assunto, é a preparação do ambiente de desenvolvimento e teste. Para isto deveremos ter as seguintes ferramentas instaladas no nosso sistema host:

- Compilador C;
- Montador ASM;
- Linkeditor;
- Utilitário MAKE;
- Emulador de Máquina Virtual.

## Compilador C

Para a implementação da maior parte do sistema utilizaremos a **Linguagem C**. Embora seja possível a utilização de diversas outras linguagens, inclusive a minha preferência pessoal seja a utilização de outra linguagem nada convencional para isto, a implementação em C é a forma *"mais padrão"* de se desenvolver um sistema operacional.

Para a compilação do código escrito em C utilizaremos o compilador **gcc**. Verifique se o mesmo está instalado no seu sistema:

```
$ which gcc
```

Uma resposta semelhante a esta indica que o mesmo já se encontra instalado:

```
/usr/bin/gcc
```

Caso não obtenha resposta, deverá instalar o gcc através do seguinte comando:
```
$ sudo apt install gcc
```

## Montador ASM


Caso não obtenha resposta, deverá instalar o nasm através do seguinte comando:

```
$ sudo apt install nasm
```

## Linkeditor

O Linkeditor é um utilitário que reune os arquivos objetos gerados durante a compilação em um arquivo executável final.

Para a montagem utilizaremos o **ld** do pacote **binutils**, que normalmente já vem instalado em todas as ditribuições linux. Contudo verifique se o mesmo está instalado no seu sistema:

```
$ which ld
```

Uma resposta semelhante a esta indica que o mesmo já se encontra instalado:

```
/usr/bin/ld
```

Caso não obtenha resposta, deverá instalar o nasm através do seguinte comando:

```
$ sudo apt install binutils
```

## Emulador de Máquina Virtual

Para o teste e depuração do sistema iremos utilizar o emulador QEMU.

Verifique se o mesmo está instalado no seu sistema:

```
$ which qemu-system-x86_64
```

se o sistema host for 64 bis, ou, se o sistema host for 32 bits:

```
$ which qemu-system-i386
```

Uma resposta semelhante a esta indica que o mesmo já se encontra instalado:

```
/usr/bin/qemu-system-x86_64
```

ou 

```
/usr/bin/qemu-system-i386
```

Caso não obtenha resposta, deverá instalar o nasm através do seguinte comando:

```
$ sudo apt install qemu
```

## Utilitários Adicionais

### GIT

Muito provavelmente você já está visualizando isto atraveś do **github**, e para o acompanhamento de todas as fases estarei utilizando as funcionalidades do utilitário **git**, que é uma ferramenta de controle de versões para código fonte.

Uma resposta semelhante a esta indica que o mesmo já se encontra instalado:

```
/usr/bin/git
```

Caso não obtenha resposta, deverá instalar o nasm através do seguinte comando:

```
$ sudo apt install git
```

#### Obtenção deste Repositório

Após a instalação do git devemos fazer o **clone** deste repositório com o comando:

```
$ git clone https://github.com/LucGoncalez/SOS.git SimplesOS
```

#### Navegando entre os códigos do tutorial

Todos os Capítulos deste Tutorial podem ser lidos a partir do último ***commit*** do repositório, contudo é necessário retonar a cada tutorial na linha de tempo que o código em questão possa ser observado, pois o mesmo irá evoluir a cada novo capítulo.

Para a navegação entre os capítulos é necessario fazer o **checkout** da respectiva ***tag*** através do **git**:

```
$ git checkout <nome_da_tag>
```

Para retornar ao último commit devemos fazer o **checkout** para o branch **main**:

```
$ git checkout main
```

Para se obter uma ***lista das tags*** disponíveis basta utilizar o seguinte comando:

```
$ git tag
```
