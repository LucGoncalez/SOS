# Tutorial 2 - Configurando Ambiente

Para o desenvolvimento de Sistemas Operacionais devemos realizar testes constantemente. A melhor e mais fácil forma de se testar um SO é com a utilização de um emulador de máquinas virtuais como disse no capítulo anterior.

Neste tutorial iremos configurar o ambiente de testes do nosso SO. O ambiente de execução deve ser confiável, ou seja, primeiro devemos ter a certeza que o código, depois de compílado, será realmente carregado da forma correta.

Aqui utilizaremos o QEMU e para a configuração do mesmo utilizaremos uma imagem de um sistema operacional conhecido e já funcional. Para isto utilizaremos a imagem de disquete de uma versão do MS-DOS que pode ser obtida facilmente na Internet.

Após baixar a imagem de disquete do MS-DOS, que possui 1.44 MB de tamanho, a mesma deverá ser renomeada para **dos.img** e colocada no diretório **tests**.

## Criação do Makefile

Para facilitar a construção e teste iremos utilizar um **Makefile**, que é um arquivo utilizado pelo **make**, que contêm instruções para a construção ou execução de algum comando.

No arquivo Makefile teremos as diretivas que podem ser chamadas pelo comando **make** diretamente:

```
$ make <diretiva>
```
